# Identity for an Agent in a Container

Proof of concept: Give an  agent a cryptographic identity - the same idea as SPIRE/SPIFFE for containers.

The "agent" folder contains the bindings to the SPIRE agent. (Similar for the "server" folder)

The workload just sleeps to show a service running in a container as a server.
